# Contributing to LAW-GPT

First off, thank you for considering contributing to LAW-GPT! It's people like you that make LAW-GPT such a powerful and expanding tool for the legal and AI communities.

## Where do I go from here?

If you've noticed a bug or have a question, [search the issue tracker](https://github.com/Knnivedh/LAW-GPT/issues) to see if someone else in the community has already created a ticket. If not, go ahead and [make one](https://github.com/Knnivedh/LAW-GPT/issues/new)!

## How can I contribute?

### 1. Improve Documentation
Whether it's fixing typos, adding new examples, or explaining the RAG architecture more clearly, documentation is always critical.

### 2. Submit Pull Requests
1. Fork the repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

## Development Setup
Please read the `README.md` for local setup instructions involving the Python fastAPI backend and Vite/React frontend. Ensure you test your changes locally against our 156k+ data points target accuracy standard before submitting a PR.
